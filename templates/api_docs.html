Introduction

The ATP Rankings API provides programmatic access to historical ATP tennis rankings data spanning from 1973 to present. All endpoints return JSON responses and require no authentication.

Base URL: {{ request.url.scheme }}://{{ request.url.netloc }}

Features

  • Access over 2,600 weeks of ATP rankings data
  • Retrieve complete rankings for any specific week
  • JSON response format for easy integration
  • No rate limiting or authentication required
  • CORS enabled for browser-based applications

API Endpoints

Get All Available Weeks

GET /api/weeks
Returns a list of all available weeks (dates) for which rankings data is available. Weeks are returned in descending chronological order (newest first).
Response Example:
{
  "weeks": [
    "2025-04-21",
    "2025-04-14",
    "2025-04-07",
    "2025-03-31",
    ...
    "1973-09-03",
    "1973-08-27"
  ]
}

Get Rankings for a Specific Week

GET /api/week/{week_date}
Returns the complete ATP rankings for a specific week, including rank, player name, and points for all ranked players.
Path Parameters:
week_date (string, required)
The date of the week in YYYY-MM-DD format (e.g., "2023-01-02")
Response Example:
{
  "week": "2023-01-02",
  "rankings": [
    {
      "rank": "1",
      "name": "Carlos Alcaraz",
      "points": "6,820"
    },
    {
      "rank": "2",
      "name": "Rafael Nadal",
      "points": "6,020"
    },
    {
      "rank": "3",
      "name": "Stefanos Tsitsipas",
      "points": "5,550"
    },
    ...
  ]
}
Error Response (404):
{
  "detail": "Week 2099-01-01 not found"
}

Usage Examples

JavaScript (Fetch API)

// Get all available weeks
fetch('{{ request.url.scheme }}://{{ request.url.netloc }}/api/weeks')
  .then(response => response.json())
  .then(data => {
    console.log('Total weeks:', data.weeks.length);
    console.log('Latest week:', data.weeks[0]);
  });

// Get rankings for a specific week
fetch('{{ request.url.scheme }}://{{ request.url.netloc }}/api/week/2023-01-02')
  .then(response => response.json())
  .then(data => {
    console.log('Week:', data.week);
    console.log('Number 1:', data.rankings[0]);
  });

Python (Requests)

import requests

# Get all available weeks
response = requests.get('{{ request.url.scheme }}://{{ request.url.netloc }}/api/weeks')
data = response.json()
print(f"Total weeks: {len(data['weeks'])}")

# Get rankings for a specific week
response = requests.get('{{ request.url.scheme }}://{{ request.url.netloc }}/api/week/2023-01-02')
data = response.json()
print(f"Week: {data['week']}")
print(f"Number 1: {data['rankings'][0]['name']}")

cURL

# Get all weeks
curl {{ request.url.scheme }}://{{ request.url.netloc }}/api/weeks

# Get specific week
curl {{ request.url.scheme }}://{{ request.url.netloc }}/api/week/2023-01-02

Response Format

Content Type

All responses are returned in JSON format with Content-Type: application/json

Status Codes

  • 200 OK - Request successful
  • 404 Not Found - Week not found in database
  • 500 Internal Server Error - Server error occurred

Data Structure

Week Data Object: 

{
  "rank": "string",     // Player's ranking position (e.g., "1", "2", "3")
  "name": "string",     // Player's full name
  "points": "string"    // Points with comma separators or "-" if not available
}

‚ú® Best Practices

  • Caching: Consider caching responses as historical data rarely changes
  • Error Handling: Always check for 404 errors when requesting specific weeks
  • Date Format: Use YYYY-MM-DD format for week dates (e.g., "2023-01-02")
  • Pagination: The API returns complete datasets - implement client-side pagination if needed
  • Rate Limiting: While there's no enforced rate limit, please be respectful with request frequency
